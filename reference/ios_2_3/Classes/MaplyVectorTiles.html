Inherits from NSObject
Conforms to MaplyPagingDelegate
Declared in MaplyVectorTiles.h

Overview

A Maply Vector Tiles database contains tiled vector data that can be displayed using styles contained within the database. The database will act as a paging delegate and is usually hooked up to a MaplyPagingLayer to do its thing.

StartRemoteVectorTiles:cacheDir:viewC:block:

Kick off a maply vector tiles database from a remote URL.

+ (void)StartRemoteVectorTiles:(NSString *)jsonURL cacheDir:(NSString *)cacheDir viewC:(MaplyBaseViewController *)viewC block:(void ( ^ ) ( MaplyVectorTiles *vecTiles ))callbackBlock

Parameters

jsonURL

The URL for the top level JSON description of this whole mess.

cacheDir

A path to the cache directory.

viewC

The view controller to create the objects in.

callbackBlock

The block that gets called (on the main thread) when the construction succeeds or fails.

Discussion

Creating a vector tiles database from a remote URL is a multi-stage process. It requires a couple of network fetches first before we can safely start the db. This runs through that process, creating the tile source and then calling the block when it’s done. It’s up to the caller to provide a block that creates the paging layer and adds it to the view controller.
